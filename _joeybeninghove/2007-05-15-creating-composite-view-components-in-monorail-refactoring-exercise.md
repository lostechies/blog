Notice the design decisions I&#8217;ve made in this one statement.&nbsp; I&#8217;ve said that I want to call a new method named **RenderComponent** which will take in an instance of&nbsp;a view component and a string array of key/value pairs as component parameters to use when rendering this view component to html.

(Side Note: The Castle MonoRail framework provides a great little utility class&nbsp;named&nbsp;**DictHelper**&nbsp;for converting an array of string key/value pairs (&#8220;key=value&#8221;) to a&nbsp;an **IDictionary**.&nbsp; Very handy for situations like this.&nbsp; More on that in a sec.)

For now, I&#8217;ll just add a new private method in this same class named **RenderComponent** and&nbsp;move the logic needed to render the **LinkSubmitButton**&nbsp;view component into this new private method.
